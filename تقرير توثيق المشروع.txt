SYSTEM ARCHITECTURE

The system is divided into multiple layers.

CLIENT LAYER

Built with:

Next.js App Router
React Components

Pages include:

Home

Shop

Product Page

Cart

Checkout

Orders

Admin Dashboard

STATE MANAGEMENT

The application uses React Context.

Main contexts:

AuthContext
Manages authentication state and user session.

CartContext
Handles cart state.

CheckoutContext
Controls checkout flow.

LanguageContext
Handles RTL/LTR switching.

OrderTrackingContext
Handles realtime tracking updates.

API LAYER

Backend logic is implemented using Next.js Route Handlers.

API routes include:

Orders

GET /api/orders
GET /api/orders/[id]
POST /api/orders

Tracking

GET /api/order-tracking/[orderId]

Invoice

GET /api/invoice/[orderId]/pdf

Cart

GET /api/cart
POST /api/cart

These APIs communicate with Supabase using the Server Client.

SUPABASE CLIENT ARCHITECTURE

The project uses:

@supabase/ssr

Two client types are used.

Browser Client

Used inside React components.

createBrowserClient()

Server Client

Used inside API routes.

createServerClient()
DATABASE ARCHITECTURE

Main database tables:

users
orders
order_tracking_events
carts
cart_items

USERS TABLE

Primary user profile table.

Columns

id uuid PK
email text
name text
avatar_url text
providers text[]
is_admin boolean
is_catalog_admin boolean
created_at timestamp

Relations

users.id
↓
orders.user_id
carts.user_id
ORDERS TABLE

Stores customer orders.

Columns

id uuid PK
user_id uuid FK → users.id
status text
total numeric
created_at timestamp

Relations

orders.id
↓
order_tracking_events.order_id
ORDER TRACKING EVENTS

Stores the timeline of order status changes.

Columns

id uuid PK
order_id uuid FK → orders.id
status text
created_at timestamp

CARTS TABLE

Stores carts per user.

Columns

id uuid PK
user_id uuid FK → users.id
created_at timestamp

CART ITEMS TABLE

Items inside carts.

Columns

id uuid PK
cart_id uuid FK → carts.id
product_id uuid
quantity integer

ROW LEVEL SECURITY (RLS)

RLS is enabled.

users

Policy

Users can read their own profile.

auth.uid() = id

orders

Policy

Users can read their own orders.

auth.uid() = user_id

cart_items

Users can modify only items belonging to their cart.

Service Role is used only for server-side operations.

USER CREATION ARCHITECTURE

User creation is handled via database trigger.

Flow

Supabase Auth
↓
auth.users
↓
Trigger
↓
public.users

This ensures:

no duplicate users

no race conditions

consistent user records

TRIGGER

Trigger Name

on_auth_user_created

Event

AFTER INSERT ON auth.users

Function

handle_new_user()

The function copies auth user data into public.users.

ORDER LIFECYCLE

Order statuses follow this lifecycle.

requested

confirmed

preparing

shipped

delivered

canceled

Each change creates a record in:

order_tracking_events

REALTIME SYSTEM

Realtime channel

order-tracking:{orderId}

Clients subscribe to:

order_tracking_events INSERT

Helper used by frontend:

subscribeToOrderTrackingEvents(orderId)

This allows live updates of order status.
